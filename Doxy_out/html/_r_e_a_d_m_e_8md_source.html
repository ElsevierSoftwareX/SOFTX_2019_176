25 ## 1. Introduction <a name="introduction"></a>
26 
27 Bembel is the
28 Boundary Element Method Based Engineering Library
29 written in C and C++ to solve boundary value problems governed by the Laplace,
30 Helmholtz or electric wave equation within the isogeometric framework [3,4,5,6].
31 It was developed as part of a cooperation between the TU Darmstadt and the
32 University of Basel, coordinated by [H. Harbrecht](#HeHa), [S. Kurz](#SK)
33 and [S. Schöps](#SSc). The code is based on the Laplace BEM of [J. Dölz](#JD),
34  [H. Harbrecht](#HeHa), and [M. Multerer](#MM), [2,7] as well as the spline
35 and geometry framework of [F. Wolf](#FW). The code was extended by
36 [J. Dölz](#JD) and [F. Wolf](#FW) in early 2018 to cover electromagnetic
37 applications [5,6].
38 A short introduction to the code has been written [3].
39 If you utilise our code as part of a publication, we would appreciate it if you cite it.
40 
41 This code is currently at version 1.0. Additional features are planned, and developement is ongoing.

53 ## 3. Features <a name="features"></a>
54 
55 Current key features include
56 
57 * Header-only implementation,
58 * Already implemented Laplace, Helmholtz and Maxwell single layer operator,
59 easily expendable to other operators,
60 * Arbitrary parametric mappings for the geometry representation, by default
61 realized as NURBS-mappings from files generated by the
62 [NURBS package](https://octave.sourceforge.io/nurbs/),
63 * Arbitrary-order B-Spline functions as Ansatz spaces, as in
64 the framework of isogeometric analysis for electromagnetics [1,5],
65 * An embedded interpolation-based fast multipole method for compression [2,5],
66 equivalent to the H2 matrix format,
67 * openMP parallelized matrix assembly,
68 * Full compatibility with the [Eigen](http://eigen.tuxfamily.org/) linear algebra library.
69 If you are not familiar with Eigen3, we emphasize that you can use it similarly
70 to Matlab or Octave,
71 see [here](http://eigen.tuxfamily.org/dox/AsciiQuickReference.txt).

73 ## 4. How to Run our Code <a name="example"></a>
74 
75 You need to install the Eigen3 library, see [Eigen's Documentation](https://eigen.tuxfamily.org/dox/GettingStarted.html) for help.
76 We do not rely on any other external libraries, except for the standard template
77 library. Thus, having installed Eigen, Bembel should run out of the box. If you want to use
78 Bembel as part of your application, simply add the `Bembel/` directory to your includes.
79 
80 If you want to run the provided examples and tests, you can utilize the procided `CMakeLists.txt`.
81 We recommend setting up a `build/` directory
82 in the root directory of the Bembel, to switch to it and then to call `cmake ..` and `make debug`
83 or `make release`, respectively.
84 
85 The CMake-File checks for installations of Eigen via the corresponding ` Eigen3Config.cmake`.
86 On Unix you may run
87 `apt install libeigen3-dev`, or on Mac with Homebrew `brew install eigen`, and everything should work. Alternatively, you can delete the line
88 ``` find_package (Eigen3 3.3 REQUIRED NO_MODULE)```
89 and all lines of the type
90 `target_link_librarires(... Eigen3::Eigen)`
91 from `CMakeLists.txt`, and give a path to the eigen headers as an include directory manually.
92 
93 Then, the examples and tests should compile without any issues. You may run all tests by calling
94 `make test` from the `build/` directory after a successful compilation.

96 ## 5. Structure of the Repository <a name="structure"></a>
97 
98 The general structure of the repository looks as follows.
99 
100 * The root directory contains some helpful shell scripts.
101 * `assets/` only contains things relevant for GitHub pages.
102 * `geo/` contains geometry files in the format of the octave
103 [NURBS package](https://octave.sourceforge.io/nurbs/). They can be utilized for computations. Note that
104 **the normal vector must be outward directed** on all patches!
105  * `geo/octave_example/` includes `.m` files that showcase how geometries can be
106  constructed using the [NURBS package](https://octave.sourceforge.io/nurbs/)
107  of octave.
108 * `Bembel/` contains the library. In `Bembel/`, module files have been created analogously to the
109 design of the Eigen library. If Bembel is used, only these module files should be included. These modules are the following:
110  * `AnsatzSpace` contains the files whose routines manage the discrete space on the surface of the geometry. Specifically, this is realised through the four classes `Superspace`, `Projector`, `Glue`, and `AnsatzSpace`. Therein, the `Superspace` manages local polynomial bases on every element. Through a transformation matrix generated by the template class `Projector` which depends on the `LinearOperator` and its defined traits, the `Superspace` can be related to B-Spline bases on every patch. To build conforming spaces (in the case of `DifferentialForm::DivergenceConforming` through continuity of the normal component across patch interfaces, and in the case of `DifferentialForm::Continuous` through global C0-continuity), the template class `Glue` assembles another transformation matrix to identify degrees of freedom across edges. Then, a coefficient vector in the `Superspace` can be related to one of the smooth B-Spline basis, as explained in [4,5].
111  * `ClusterTree` implements an element structure for refinement. Currently, only uniform refinement has been implemeneted. A `ClusterTree` object itself is comprised of a `Geometry` object and an `ElementTree`. The `ElementTree` provides routines for the extraction of topological information, refinement, as well as iterators to effortlessly loop over elements.
112  * `DuffyTrick` provides quadrature routines for (nearly) singular integrals. Therein, quadrature routines for 4 cases are implemented: Separated elements, elements sharing a corner, elements sharing an edge, and identical elements.
113  * `DummyOperator` provides a `LinearOperator` for testing.
114  * `Geometry` implements the handling of the geometry.
115  * `H2Matrix` handles the compression of the large, densely populated matrices. The algorithm was introduced in [2] and was generalized to the Maxwell case in [5].
116  * `Helmholtz` provides the necessary specializations to solve Helmholtz problems.
117  * `IO` provides input-ouput functionality, including routines for VTK file export, timing, and writing log files.
118  * `Laplace` provides the necessary specializations to solve Laplace problems.
119  * `LinearForm` implements trace operators, i.e., routines to generate the right hand side of the linear systems. Currently, only a Dirichlet trace and a rotated tangential trace are provided.
120  * `LinearOperator` provides a framework to implement lienar operators that can be used to solve PDEs. Therein, the kernel function, a routine for the evaluation, and certain traits (like the scalar type, the operator order, and the `DifferentialForm`, i.e., weather the basis must be discontinuous, vector-valued and divergence conforming, or globally continuous) required for discretization must be provided.
121  * `Maxwell` provides the necessary specializations to solve Maxwell problems.
122  * `Potential` provides routines for the evaluation of the `Potential`, i.e., the solution to the PDE via the solution of (non-singular) integrals based on the `LinearOperator`s.
123  * `Quadrature`
124  * `Spline` provied basic routines related to spline function and local polynomials.
125 
126 The `Bembel/src/` directory leads to folders corresponding to these modules, which include the actual implementation.
127 The folder `examples/` contains short and *well documented* examples for Bembels functionality and `tests/` contains the tests for Bembel.
